# 우리 가족 채팅 MVP

초대 기반으로만 입장하는 폐쇄형 가족 채팅 PWA입니다. 친구 검색이나 전화번호 기반 자유 연결 없이, 가족 그룹 안에서만 가족 전체방과 1:1 대화를 허용하는 구조를 우선 반영했습니다.

## 현재 구현 범위

- 관리자만 가족 그룹 생성 가능
- 관리자만 초대 코드 생성 가능
- 초대 코드 기반 가족 참여
- 가족 전체 공용 채팅방 1개
- 가족 내부 1:1 채팅 자동 생성
- 텍스트 메시지 저장
- 이미지 첨부 전송
- 읽음 상태 기본 표시
- 최근 메시지 미리보기 / 채팅방 목록
- 마지막 활동 시간 표시
- 모바일 중심 UI
- PWA manifest / 서비스 워커 / 설치 버튼 / 오프라인 앱 셸 캐시
- 방별 알림 on/off 토글

## 닫힌 가족 구조 원칙

- 검색 가능한 사용자 디렉터리가 없습니다.
- 전화번호 기반 연결이 없습니다.
- 초대 코드를 알고 있어도 해당 가족 그룹 안으로만 들어갑니다.
- 1:1 방은 같은 가족 내부 멤버끼리만 생성됩니다.
- 그룹 외부 사용자와 연결되는 UI를 두지 않았습니다.
- 관리자 권한은 가족 생성 시에만 부여되며 일반 사용자는 관리자 승격 기능이 없습니다.

## 실행 방법

정적 파일만으로 동작합니다. 서비스 워커와 설치 기능을 보려면 `file://` 대신 로컬 정적 서버로 열어야 합니다.

예시:

```powershell
npx serve .
```

## 현재 코드의 성격

이 저장소는 실행 가능한 모바일 PWA 데모를 우선 제공합니다. 로컬 저장소(`localStorage`)와 `BroadcastChannel`을 사용하므로 같은 브라우저의 여러 탭 테스트에는 적합하지만, 실제 여러 기기 간 동기화까지는 처리하지 않습니다.

즉:

- UI/도메인 규칙/PWA 흐름은 바로 확인 가능
- 다중 기기 실시간 동기화, 서버 검증, 푸시 알림은 프로덕션 백엔드가 필요

## 프로덕션 권장 아키텍처

- Web App / PWA: 현재 프런트엔드 구조 유지
- Auth: 관리자 생성 초대 토큰 + 기기 세션 토큰
- API: Cloudflare Workers 또는 동급의 서버리스 API
- 실시간: 가족 단위 또는 방 단위 WebSocket / Durable Object
- DB: 가족, 멤버, 초대, 방, 메시지, 읽음 상태는 D1/Postgres류
- 이미지: R2/S3류 오브젝트 스토리지
- 푸시: Web Push + 모바일 브라우저 지원 범위별 분기

핵심 서버 규칙:

- 모든 요청은 `family_id`로 스코프 강제
- 초대 토큰은 단일 가족에만 매핑
- DM 방은 `(family_id, member_a, member_b)` 조합으로만 생성
- 메시지 조회, 업로드, 알림 모두 가족 범위를 벗어나면 즉시 거부

## 2차 확장 권장 항목

- 파일 전송 고도화
- 읽음 상태 상세화
- Web Push 알림
- 음성/화상 통화
- 소규모 가족 그룹방
- 관리자 대시보드
- 가족별 테마/브랜딩
